
	[게시글 수정]

	1. 요구사항 요소 확인

	1) 데이터 요구:
	- 게시글 table(num, title, author, content, wdate, hitcnt)
	- 게시글 table에서 PK는 num
	- 게시글 table에서 num, title, author은 not null
	
	2) 기능 요구: 게시글 수정
	- 게시글 확인 화면에서 게시글 수정 버튼을 누른다
	- 게시글을 수정할 수 있는 화면으로 이동한다
	- 게시글 제목과 내용을 수정한다	
	- 페이지 하단의 버튼을 누른다
		- 확인 버튼을 눌러 게시글 내용을 수정한다
			- 확인 버튼을 누르면 게시글 수정 여부를 한번 더 확인할 수 있도록 알림창을 띄운다
				- 확인을 누르면 변경된 내용을 저장하고 확인한다
				- 취소를 누르면 기존의 수정 화면으로 이동한다
		- 취소 버튼을 눌러 변경된 내용을 리셋한다
		- 목록버기 버튼을 눌러 게시글 리스트로 돌아간다

	3) 제품, 서비스 품질
	
	4) 제약사항:


	2. 정황 시나리오 작성(사용자 관점)
	- 사용자가 게시글 리스트 화면에서 수정할 게시글의 제목을 누르면 게시글 내용 화면으로 이동한다
	- 사용자가 게시글 내용 화면 우측 하단의 수정 버튼을 누르면 게시글 수정 화면으로 이동한다
	- 사용자는 게시글 수정 화면의 게시글 번호, 제목, 작성자(1행), 작성일, 조회수(2행), 게시글 내용(3행) 중 게시글 제목과 내용을 수정할 수 있다
	- 사용자가 게시글 수정 화면 좌측 하단의 목록보기 버튼을 누르면 게시글 리스트 화면으로 이동한다
	- 사용자가 게시글 수정 화면에서 우측 하단의 저장 버튼을 누르면 저장 확인, 취소를 확인하는 알림창이 뜨며, 
	  확인을 누르면 수정한 내용이 반영된 게시글 내용 화면으로 이동하고, 취소를 누르면 기존 내용이 반영된 게시글 내용 화면으로 이동한다
	- 사용자가 게시글 수정 화면 우측 하단의 취소 버튼을 누르면 수정한 게시글 제목과 내용이 기존 내용으로 리셋된다
